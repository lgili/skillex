## Overview

This document provides a foundational understanding of Obsidian's core concepts. It's intended for users who are new to Obsidian or need a refresher on the basics.

## Key Concepts

- **Vault**: A vault is a folder on your local filesystem where Obsidian stores your notes, attachments, and configuration files. You can have multiple vaults for different purposes (e.g., work, personal).
- **Notes**: Notes are the fundamental building blocks of Obsidian. They are plain text files written in Markdown.
- **Markdown**: A lightweight markup language for creating formatted text using a plain-text editor. Obsidian uses Markdown for all notes.
- **Links**: Obsidian allows you to link notes together, creating a network of knowledge. This is a core feature that enables the "second brain" concept.
- **Graph View**: A visual representation of the connections between your notes. It helps you discover relationships and navigate your knowledge base.
- **Plugins**: Obsidian has a rich ecosystem of community-developed plugins that extend its functionality.

## Examples

### Creating a new note

1.  Open Obsidian.
2.  Click the "New note" button in the sidebar, or use the `Ctrl/Cmd+N` keyboard shortcut.
3.  Give your note a title.
4.  Start writing in Markdown.

### Linking to another note

To create a link to another note, simply type `[[` and the name of the note you want to link to. For example: `[[My other note]]`.

### Basic Markdown Syntax

```markdown
# This is a heading 1
## This is a heading 2

- This is a bulleted list
- another item

1. This is a numbered list
2. another item

*This is italic text*
**This is bold text**
```

## Anti-Patterns

- **Creating overly complex folder structures**: Start with a simple structure and let it evolve organically.
- **Not using links**: The power of Obsidian comes from the connections between notes. Make sure to link your notes together.
- **Installing too many plugins at once**: Start with the core features and only add plugins as you need them.
- **Forgetting to back up your vault**: Your vault is a folder of plain text files, so it's easy to back up. Use a cloud storage service or a version control system like Git.
